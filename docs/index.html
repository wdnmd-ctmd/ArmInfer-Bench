诚实说明

  • NF4:结论只用同机 speedup ratio,不同 job/runner 绝对值不可直接对比。
  • TTFT:推算(512 prefill),公式 ttft_ms = pp_n / prefill_tok_s × 1000,非实测首 token。
  • PPL:误差棒 ±0.64~0.68 重叠,排序可信但非精确质量差;量化取舍主由体积/速度/内存驱动。
  • G5:decode 差 <5% 取峰值内存更低者(内存换速度取舍)。
  • naive:armv8-a 基线,仍含 NEON(非零 SIMD),不开 i8mm/KleidiAI/repack。
  • Serving (T4b):可选维度,失败/超时不影响 15 速度 JSON(M1 拆两次 commit,G4)。数据缺失时本区段显示占位。吞吐口径 = 总 token / wall-clock(S1),非各请求吞吐之和。
